업로드해주신 두 번째 문서(상하이교통대학의 "프롬프트 학습과 생각의 사슬(Prompt Learning and Chain of Thought)")의 핵심 내용을 요약해 드립니다. 

이 문서는 대규모 언어 모델(LLM)을 효과적으로 활용하기 위해 지시어를 설계하는 '프롬프트 엔지니어링'의 기초부터 고급 기법(특히 생각의 사슬)까지 다루고 있습니다.

**1. 프롬프트 학습의 개념과 필요성**
* [cite_start]**등장 배경**: 매개변수가 방대한 대형 언어 모델(LLM)을 특정 작업마다 미세조정(Fine-tuning)하는 것은 막대한 컴퓨팅 비용이 발생합니다[cite: 110, 111]. 
* [cite_start]**개념**: 모델의 가중치를 수정(미세조정)하지 않고도, 자연어 프롬프트만으로 사전 훈련된 모델의 패턴을 유도하여 다운스트림(Downstream) 작업을 수행하는 방식입니다[cite: 112, 113].
* [cite_start]**장단점**: 직관적인 자연어 사용으로 유연성이 높고 확장이 쉽다는 장점이 있지만 [cite: 143, 144, 145][cite_start], 사용자가 얼마나 명확하게 표현하는지에 따라 결과가 달라지며 복잡한 작업은 여러 개의 프롬프트를 이어야(체인) 하는 단점도 있습니다[cite: 147, 148].

**2. 프롬프트 엔지니어링의 4대 구성 요소**
좋은 프롬프트는 일반적으로 다음 요소들로 구성됩니다.
* [cite_start]**지시 (Instruction)**: 모델이 해야 할 작업을 명확히 지시 ("~를 요약해 줘")[cite: 279].
* [cite_start]**문맥 (Context)**: 작업의 배경 정보나 부가적인 조건[cite: 280].
* [cite_start]**입력 데이터 (Input Data)**: 처리가 필요한 핵심 데이터 정보[cite: 281].
* [cite_start]**출력 지시 (Output Indicator)**: 모델이 답변을 생성할 위치나 반환 형식[cite: 282].

**3. 주요 프롬프트 기법**
* [cite_start]**제로샷 프롬프트 (Zero-shot Prompting)**: 예시 없이 지시만으로 작업을 수행하는 방식입니다[cite: 16]. [cite_start]단계별로 생각하도록 명시하거나 특정 출력 형식(예: JSON)을 지정하여 품질을 높일 수 있습니다[cite: 23, 24]. [cite_start]때로는 모델에게 중요성을 강조하거나 감정적인 호소(Emotional Prompting)를 더하거나, 온도(Temperature) 등의 하이퍼파라미터를 조절하여 원하는 결과를 유도하기도 합니다[cite: 27, 28].
* [cite_start]**퓨샷 프롬프트 (Few-shot Prompting)**: 모델에게 소수의 '입력-출력 예시'를 제공하여 모델이 작업의 패턴을 학습하도록 돕는 기법입니다[cite: 33]. [cite_start]이때 예시의 개수, 일관된 형식, 예시와 실제 문제 간의 관련성 등이 성능에 큰 영향을 미칩니다[cite: 34].

**4. 생각의 사슬 (Chain of Thought, CoT)**
* [cite_start]**개념**: 수학이나 복잡한 논리 추론 문제를 풀 때, 모델이 사람처럼 '단계별로 생각'하도록 유도하여 정답률을 획기적으로 높이는 기술입니다[cite: 47].
* [cite_start]**작동 방식**: 기존의 단순한 <문제 → 정답> 방식이 아니라, <문제 → 생각의 사슬(중간 풀이 과정) → 정답> 순서로 도출하게 하여 추론의 투명성과 신뢰성을 높입니다[cite: 47].
* [cite_start]**제로샷 CoT**: 질문 뒤에 "우리 한 번 단계별로 차근차근 생각해보자(Let's think step by step)"라는 문구를 붙이는 것만으로도 모델의 숨겨진 추론 능력을 이끌어낼 수 있습니다[cite: 49].

**5. 고급 프롬프트 기술**
[cite_start]단순한 텍스트 생성을 넘어, 외부 지식을 검색해 답변하는 **검색 증강 생성(RAG)**, 스스로 논리적 추론과 외부 도구 사용 행동을 반복하는 **ReAct**, 자신의 실수를 되돌아보고 답변을 수정하는 **자기 반성(Self-Reflection)** 등 최신 프롬프팅 기법들로 발전하고 있습니다[cite: 18, 19].
