# Chapter 7: LLM 기반 텍스트 스테가노그래피 (Text Steganography)

이 문서는 대규모 언어 모델(LLM)을 활용한 텍스트 스테가노그래피의 핵심 개념과 기법을 정리합니다.

## 1. 스테가노그래피란?

### 1.1 정의

스테가노그래피(Steganography)는 그리스어 "steganos(숨겨진)"와 "graphein(쓰다)"에서 유래한 용어로, 비밀 메시지의 존재 자체를 숨기는 정보 은닉 기술입니다. 일반적인 커버 매체(텍스트, 이미지, 오디오 등) 안에 비밀 정보를 삽입하여, 제3자가 보기에는 평범한 데이터처럼 보이도록 합니다.

### 1.2 암호화(Cryptography)와의 차이

| 구분 | 암호화 (Cryptography) | 스테가노그래피 (Steganography) |
|------|----------------------|-------------------------------|
| 목적 | 메시지의 **내용**을 보호 | 메시지의 **존재** 자체를 숨김 |
| 결과물 | 의미 없는 암호문 (의심 유발 가능) | 자연스러운 커버 매체 (의심 유발 없음) |
| 탐지 | 암호문의 존재는 알 수 있음 | 비밀 통신이 이루어지는지 자체를 알기 어려움 |
| 보안 초점 | 해독 불가능성 | 탐지 불가능성 (Undetectability) |

두 기술은 상호 보완적이며, 비밀 메시지를 먼저 암호화한 후 스테가노그래피로 숨기면 이중 보안을 달성할 수 있습니다.

### 1.3 스테가노그래피의 유형

- **이미지 스테가노그래피**: 이미지 픽셀의 최하위 비트(LSB)를 변경하여 정보를 숨김
- **오디오 스테가노그래피**: 오디오 신호에 비밀 정보를 삽입
- **비디오 스테가노그래피**: 비디오 프레임에 정보를 은닉
- **텍스트 스테가노그래피**: 자연어 텍스트를 생성하면서 비밀 정보를 삽입 ← 본 챕터의 주제

## 2. 텍스트 스테가노그래피

### 2.1 전통적 텍스트 스테가노그래피

초기 텍스트 스테가노그래피는 다음과 같은 방식을 사용했습니다:

- **포맷 기반**: 공백, 탭, 줄바꿈 등 보이지 않는 문자를 삽입
- **언어학 기반**: 동의어 치환을 통해 비트를 인코딩 (예: "big" → 0, "large" → 1)
- **통계 기반**: 텍스트의 통계적 특성을 조작

이러한 방식들은 은닉 용량이 낮고, 통계 분석에 의해 탐지될 수 있다는 한계가 있었습니다.

### 2.2 LLM 기반 텍스트 스테가노그래피

대규모 언어 모델의 등장으로 텍스트 스테가노그래피는 큰 발전을 이루었습니다. LLM은 자연스러운 텍스트를 생성하는 능력이 뛰어나기 때문에, 생성 과정에서 비밀 비트를 삽입하더라도 텍스트의 자연스러움을 유지할 수 있습니다.

**기본 원리:**

1. 언어 모델이 다음 토큰의 확률 분포 P(w|context)를 계산
2. 확률이 높은 상위 k개의 토큰 후보를 선택
3. 각 후보 토큰에 이진 코드를 할당
4. 숨기려는 비트열과 일치하는 코드를 가진 토큰을 선택
5. 선택된 토큰을 출력하고, 다음 단계로 진행

이 과정을 통해 생성된 텍스트는 언어 모델의 확률 분포를 따르므로 자연스러우면서도, 각 토큰 선택에 비밀 비트가 인코딩되어 있습니다.

## 3. 핵심 코딩 방식

### 3.1 허프만 코딩 (Huffman Coding)

허프만 코딩은 1952년 David Huffman이 제안한 최적 접두사 코드(prefix code) 알고리즘입니다.

**원리:**
- 빈도(확률)가 높은 심볼에는 짧은 코드를, 낮은 심볼에는 긴 코드를 할당
- 접두사 자유(prefix-free) 특성: 어떤 코드도 다른 코드의 접두사가 되지 않아 모호함 없이 디코딩 가능

**허프만 트리 구성 과정:**
1. 각 토큰을 확률값을 가진 리프 노드로 생성
2. 가장 작은 확률을 가진 두 노드를 선택하여 합침
3. 합쳐진 노드의 확률은 두 자식 노드 확률의 합
4. 노드가 하나만 남을 때까지 반복
5. 루트에서 리프까지의 경로가 각 토큰의 코드 (왼쪽=0, 오른쪽=1)

**스테가노그래피에서의 활용:**
- 토큰의 예측 확률을 빈도로 사용하여 허프만 트리를 구성
- 확률이 높은 토큰일수록 짧은 코드를 가지므로, 자주 선택되어 텍스트의 자연스러움을 유지
- 가변 길이 코드 덕분에 평균적으로 높은 비트 은닉 효율을 달성

**예시:**
```
토큰 A (확률 0.4) → 코드: "0"      (1비트)
토큰 B (확률 0.3) → 코드: "10"     (2비트)
토큰 C (확률 0.2) → 코드: "110"    (3비트)
토큰 D (확률 0.1) → 코드: "111"    (3비트)
```

### 3.2 고정 길이 코딩 (Fixed-Length Coding, FLC)

고정 길이 코딩은 모든 토큰 후보에 동일한 길이의 이진 코드를 할당하는 방식입니다.

**원리:**
- 상위 2^k개의 토큰을 선택
- 각 토큰에 k비트의 고정 길이 코드를 순서대로 할당
- 매 생성 단계마다 정확히 k비트를 인코딩

**예시 (k=2, 상위 4개 토큰):**
```
토큰 A (확률 0.4) → 코드: "00"
토큰 B (확률 0.3) → 코드: "01"
토큰 C (확률 0.2) → 코드: "10"
토큰 D (확률 0.1) → 코드: "11"
```

**장단점:**
- 장점: 구현이 간단하고, 매 단계 고정된 비트 수를 인코딩하여 예측 가능
- 단점: 확률이 낮은 토큰도 동일한 빈도로 선택될 수 있어 텍스트 품질이 저하될 수 있음

### 3.3 두 방식의 비교

| 구분 | 허프만 코딩 | 고정 길이 코딩 (FLC) |
|------|-----------|---------------------|
| 코드 길이 | 가변 (확률에 비례) | 고정 (k비트) |
| 은닉 효율 | 높음 (엔트로피에 근접) | 보통 |
| 텍스트 품질 | 높음 (확률 분포 유지) | 상대적으로 낮음 |
| 구현 복잡도 | 높음 (트리 구성 필요) | 낮음 |
| 단계당 은닉 비트 | 가변적 | 고정 (k비트) |

## 4. 주요 평가 지표

### 4.1 은닉 용량 효율 (Embedding Capacity)

토큰(또는 단어) 하나당 평균적으로 숨길 수 있는 비밀 비트 수를 나타냅니다. 단위는 주로 bpw(Bits Per Word)를 사용합니다.

- 허프만 코딩의 이론적 상한 보장: 텍스트 확률 분포의 섀넌 엔트로피(Shannon Entropy) $H(P)$에 근접하게 비트를 숨길 수 있습니다.
- FLC의 은닉 용량: 코드를 $k$ 비트로 설정 시 정확히 $k$ bpw가 됩니다.

### 4.2 텍스트 품질 (Text Quality)

스테가노그래피 텍스트가 일반 텍스트와 비교해 얼마나 자연스러운지를 평가합니다:

- **퍼플렉시티(Perplexity)**: 언어 모델이 생성된 텍스트 계열을 얼마나 잘 예측하는지 측정합니다. 수치가 낮을수록 텍스트가 자연스럽고 유창함을 의미합니다.
- **KL 발산(KL Divergence)**: 스테가노그래피가 적용된 텍스트의 확률 분포와 일반 텍스트 확률 분포 간의 괴리도를 뜻합니다. 이 값이 0에 가까울수록 두 텍스트가 통계적으로 구별 불가능하여 탐지가 어려워집니다.

### 4.3 탐지 불가능성 (Undetectability)

스테가노그래피 분석(Steganalysis)에 의해 탐지되지 않는 정도를 평가합니다. 이상적인 스테가노그래피 시스템은 생성된 텍스트의 통계적 분포가 일반 텍스트와 구별 불가능해야 합니다.

## 5. 실습 코드 구조 (`llm_stega_ko.ipynb`)

실습 노트북에서는 다음과 같은 구조로 LLM 기반 스테가노그래피를 구현합니다:

1. **환경 설정**: PyTorch, Transformers 라이브러리 임포트
2. **허프만 트리 구현**: `Node`, `createHuffmanTree`, `huffmanEncoding` 함수
3. **스테가노그래피 핸들러**:
   - `Huffman` 클래스: 허프만 코딩 기반 인코딩/디코딩
   - `FLC` 클래스: 고정 길이 코딩 기반 인코딩/디코딩
4. **텍스트 생성**: GPT-2 모델을 사용하여 스테가노그래피 텍스트와 일반 텍스트를 동시에 생성
5. **결과 비교**: 두 텍스트를 비교하여 스테가노그래피의 효과를 확인

## 6. 관련 연구 및 발전 방향

### 6.1 산술 코딩 (Arithmetic Coding) 기반 방식

허프만 코딩의 한계를 극복하기 위해 산술 코딩을 활용하는 방식도 연구되고 있습니다. 산술 코딩은 전체 메시지를 하나의 실수 구간으로 인코딩하여 이론적 엔트로피에 더 가깝게 접근할 수 있습니다.

### 6.2 분포 보존 스테가노그래피 (Distribution-Preserving Steganography)

생성된 텍스트의 확률 분포를 원래 언어 모델의 분포와 완벽하게 일치시키는 방식입니다. 이론적으로 완벽한 탐지 불가능성을 달성할 수 있으며, Meteor 등의 알고리즘이 대표적입니다.

### 6.3 스테가노그래피 분석 (Steganalysis)

스테가노그래피를 탐지하는 기술도 함께 발전하고 있습니다:

- **통계 기반 탐지**: 텍스트의 통계적 특성(n-gram 분포, 엔트로피 등)을 분석
- **딥러닝 기반 탐지**: 신경망을 학습시켜 스테가노그래피 텍스트와 일반 텍스트를 분류
- **조건부 확률 분석**: 언어 모델의 예측 확률과 실제 토큰 선택 간의 불일치를 탐지

### 6.4 LLM 시대의 새로운 과제

- **대형 모델의 활용**: GPT-4, LLaMA 등 더 강력한 모델을 사용하면 텍스트 품질이 향상되어 탐지가 더 어려워짐
- **다국어 스테가노그래피**: 다양한 언어에서의 스테가노그래피 연구
- **멀티모달 스테가노그래피**: 텍스트뿐 아니라 이미지, 오디오를 함께 활용하는 방식
- **보안 통신 응용**: 검열 우회, 프라이버시 보호 등 실제 응용 분야로의 확장
