# Chapter 9: GUI 에이전트 — 대규모 언어 모델 기반 GUI 지능형 에이전트 구축

이 문서는 대규모 언어 모델(LLM)을 활용한 GUI(Graphical User Interface) 에이전트의 개념, 기술 로드맵, 그리고 자기적응형 인간-컴퓨터 상호작용 GUI 에이전트를 구축하는 방법을 정리합니다.

## 1. GUI 에이전트란?

### 1.1 정의

GUI 에이전트(GUI Agent)는 컴퓨터나 모바일 기기의 그래픽 사용자 인터페이스를 사람처럼 인식하고 조작할 수 있는 AI 시스템입니다. 화면의 스크린샷을 시각적으로 이해하고, 클릭·입력·스크롤 등의 행동을 자율적으로 수행하여 사용자가 지시한 작업을 완수합니다.

### 1.2 핵심 능력

GUI 에이전트가 갖추어야 할 핵심 능력은 다음과 같습니다:

- **시각적 이해(Visual Understanding)**: 스크린샷에서 UI 요소(버튼, 텍스트 필드, 아이콘 등)의 위치와 기능을 파악
- **작업 계획(Task Planning)**: 고수준의 사용자 지시를 단계별 하위 작업으로 분해
- **행동 실행(Action Execution)**: 클릭, 타이핑, 스크롤 등 구체적인 GUI 조작을 수행
- **상태 추적(State Tracking)**: 이전 행동의 결과를 반영하여 현재 상태를 파악하고 다음 행동을 결정

### 1.3 기존 접근 방식과의 차이

| 구분 | 전통적 자동화 (RPA 등) | GUI 에이전트 |
|------|----------------------|-------------|
| UI 인식 방식 | DOM/접근성 트리 기반 | 스크린샷 시각 인식 기반 |
| 유연성 | 사전 정의된 규칙에 의존 | 새로운 화면에도 적응 가능 |
| 일반화 | 특정 앱/화면에 한정 | 다양한 앱과 환경에 범용 적용 |
| 핵심 기술 | 규칙 기반 스크립트 | 비전-언어 모델(VLM) |

## 2. GUI 에이전트의 기술 로드맵

### 2.1 비전-언어 모델(VLM) 기반 접근

최신 GUI 에이전트는 텍스트와 이미지를 동시에 이해할 수 있는 비전-언어 모델(Vision-Language Model)을 핵심으로 활용합니다. 대표적인 모델로는 Qwen2-VL, GPT-4V, CogAgent 등이 있습니다.

**작동 흐름:**
1. 현재 화면의 스크린샷을 모델에 입력
2. 사용자의 최종 목표(task)와 이전 행동 이력(previous actions)을 텍스트로 함께 제공
3. 모델이 다음에 수행할 행동(action)을 예측하여 출력
4. 예측된 행동을 실제 기기에서 실행
5. 작업이 완료될 때까지 1~4를 반복

### 2.2 행동 공간(Action Space)

GUI 에이전트가 수행할 수 있는 행동은 크게 기본 행동과 커스텀 행동으로 나뉩니다:

**기본 행동 (Basic Actions):**
- `CLICK <point>[[x, y]]</point>`: 지정된 좌표를 클릭
- `TYPE [텍스트]`: 지정된 텍스트를 입력
- `SCROLL [방향]`: 지정된 방향(UP/DOWN/LEFT/RIGHT)으로 스크롤

**커스텀 행동 (Custom Actions):**
- `PRESS_BACK`: 이전 화면으로 돌아가기
- `PRESS_HOME`: 홈 화면으로 이동
- `ENTER`: 엔터 키 입력
- `IMPOSSIBLE`: 작업 수행 불가능을 표시

### 2.3 주요 연구 동향

- **OS-Atlas**: 대규모 GUI 그라운딩 데이터를 활용하여 범용 GUI 에이전트를 구축하는 연구
- **OS-Kairos**: 에이전트의 자기적응형 인간-컴퓨터 상호작용을 구현하여, 에이전트가 자신의 능력 한계를 인식하고 필요시 인간에게 도움을 요청하는 연구
- **CogAgent**: 고해상도 스크린샷 이해에 특화된 비전-언어 모델 기반 GUI 에이전트

## 3. 자기적응형 인간-컴퓨터 상호작용 GUI 에이전트

### 3.1 개념

자기적응형(Self-Adaptive) GUI 에이전트는 단순히 행동을 예측하는 것을 넘어, 자신의 행동에 대한 **신뢰도 점수(Confidence Score)**를 함께 출력합니다. 이를 통해 에이전트가 자신의 능력 범위를 스스로 판단하고, 신뢰도가 낮은 경우 인간의 개입을 요청할 수 있습니다.

### 3.2 신뢰도 점수 체계

| 점수 | 의미 |
|------|------|
| 5 | 이 행동이 목표를 확실히 달성할 것으로 판단 |
| 4 | 이 행동이 목표를 달성할 가능성이 매우 높음 |
| 3 | 이 행동이 목표를 달성할 가능성이 어느 정도 있음 |
| 2 | 이 행동이 목표를 달성할 가능성이 매우 낮음 |
| 1 | 이 행동으로는 목표를 달성할 수 없다고 판단 |

### 3.3 활용 시나리오

- **점수가 높은 경우 (4~5)**: 에이전트가 자율적으로 행동을 실행
- **점수가 중간인 경우 (3)**: 에이전트가 행동을 제안하되, 사용자에게 확인을 요청
- **점수가 낮은 경우 (1~2)**: 에이전트가 현재 지시가 자신의 능력 범위를 벗어남을 알리고, 인간의 직접 개입을 요청

이러한 메커니즘을 통해 완전 자율과 완전 수동 사이의 유연한 인간-AI 협업이 가능해집니다.

## 4. GUI 에이전트 구축 파이프라인

### 4.1 데이터 준비

OS-Kairos 데이터셋은 다음과 같은 구조를 가집니다:

- `task`: 사용자의 최종 목표 (예: "음악 앱을 열고 노래를 검색하여 재생")
- `image_path`: 현재 화면 스크린샷 경로
- `list`: 작업을 완수하기 위한 단계별 하위 작업 목록
- `now_step`: 현재 진행 중인 단계 번호
- `previous_actions`: 이전에 수행한 행동 이력
- `osatlas_action`: OS-Atlas 모델이 예측한 행동
- `teacher_action`: 정답(교사) 행동
- `score`: 행동의 신뢰도 점수 (1~5)
- `success`: 작업 성공 여부

### 4.2 데이터 전처리

원본 데이터를 ShareGPT 형식으로 변환하여 LLM 미세조정에 적합한 대화 형태로 가공합니다. 각 데이터 항목은 다음과 같이 변환됩니다:

- **사용자 메시지**: 행동 공간 정의 + 신뢰도 점수 기준 + 최종 목표 + 이전 행동 이력 + 스크린샷
- **어시스턴트 응답**: 예측 행동 + 신뢰도 점수

### 4.3 지도 학습 미세조정 (SFT)

LLaMa-Factory 프레임워크를 사용하여 Qwen2-VL-7B 모델을 미세조정합니다:

1. **데이터셋 등록**: `data/dataset_info.json`에 Kairos 데이터셋 정보를 추가
2. **훈련 설정**: YAML 파일에 모델 경로, 학습률, 에폭 수 등 하이퍼파라미터를 구성
3. **분산 훈련**: DeepSpeed ZeRO Stage 3을 활용한 다중 GPU 분산 훈련 실행

**주요 훈련 하이퍼파라미터:**

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| learning_rate | 1.0e-5 | 학습률 |
| num_train_epochs | 5.0 | 훈련 에폭 수 |
| per_device_train_batch_size | 2 | GPU당 배치 크기 |
| gradient_accumulation_steps | 2 | 그래디언트 누적 단계 |
| lr_scheduler_type | cosine | 학습률 스케줄러 |
| warmup_ratio | 0.1 | 워밍업 비율 |
| bf16 | true | BFloat16 혼합 정밀도 사용 |

### 4.4 추론 및 검증

훈련이 완료된 모델로 추론을 수행하는 방법:

- **LLaMa-Factory 원클릭 추론**: `llamafactory-cli webchat` 명령어로 웹 채팅 인터페이스를 통해 간편하게 추론
- **커스텀 추론**: transformers와 torch 라이브러리를 사용하여 직접 추론 코드 작성

추론 시 에이전트는 입력된 스크린샷과 작업 지시에 대해 **다음 행동(action)**과 **신뢰도 점수(score)**를 출력합니다.

## 5. 핵심 모델: Qwen2-VL-7B

### 5.1 모델 개요

Qwen2-VL-7B는 Alibaba에서 개발한 비전-언어 모델로, 텍스트와 이미지를 동시에 처리할 수 있습니다. GUI 에이전트 구축에 적합한 이유는 다음과 같습니다:

- **고해상도 이미지 이해**: 다양한 해상도의 스크린샷을 처리 가능
- **좌표 기반 그라운딩**: 이미지 내 특정 위치를 좌표로 지정하는 능력
- **다국어 지원**: 다양한 언어의 UI를 이해 가능
- **오픈소스**: 연구 및 상업적 활용이 자유로움

### 5.2 미세조정 프레임워크: LLaMa-Factory

LLaMa-Factory는 다양한 LLM의 미세조정을 통합적으로 지원하는 프레임워크입니다:

- **다양한 미세조정 방식 지원**: Full Fine-tuning, LoRA, QLoRA 등
- **멀티모달 지원**: 텍스트뿐 아니라 이미지 입력이 포함된 데이터셋 처리
- **분산 훈련**: DeepSpeed를 활용한 대규모 분산 훈련 지원
- **간편한 추론**: 웹 채팅 인터페이스를 통한 원클릭 추론

## 6. 관련 연구 및 발전 방향

### 6.1 GUI 에이전트의 주요 과제

- **일반화(Generalization)**: 학습하지 않은 새로운 앱이나 화면에서도 정확하게 동작하는 능력
- **장기 작업 수행(Long-horizon Tasks)**: 수십 단계에 걸친 복잡한 작업을 안정적으로 완수하는 능력
- **오류 복구(Error Recovery)**: 잘못된 행동을 인식하고 스스로 수정하는 능력
- **안전성(Safety)**: 의도하지 않은 위험한 조작(데이터 삭제, 결제 등)을 방지하는 메커니즘

### 6.2 발전 방향

- **크로스 플랫폼 에이전트**: 모바일, 데스크톱, 웹 등 다양한 플랫폼에서 통합적으로 동작하는 에이전트
- **멀티모달 상호작용**: 시각 정보뿐 아니라 음성, 제스처 등 다양한 입력 모달리티를 활용
- **자기 진화(Self-Evolution)**: 사용 경험을 통해 스스로 능력을 향상시키는 에이전트
- **인간-AI 협업 최적화**: 신뢰도 기반의 적응적 자율성 조절을 통한 효율적인 협업 체계 구축
